# 问题跟踪

## 问题 1：RAGFlow 下载链接仅限内网访问

**状态**：待解决

**描述**：
目前 wecom-bot v3 在企业微信回复中的文档下载链接（`http://10.18.11.77:8088/doc-download/xxx`）只有公司内网能访问。外网用户（如出差、居家办公）点击链接无法下载文档。

**原因**：
RAGFlow 服务器 IP `10.18.11.77` 是内网地址，外网不可达。

**解决方向**：

| 方案 | 说明 | 复杂度 |
|------|------|--------|
| 方案 A：RAGFlow 对外暴露 | 给 RAGFlow 配公网域名 + HTTPS，防火墙只开放 doc-download 路由 | 中等 |
| 方案 B：反向代理 | 在公网可访问的 nginx 上加一层代理，只转发 /doc-download/ 请求到内网 RAGFlow | 中等 |
| 方案 C：VPN | 外网用户先连公司 VPN，再访问内网链接 | 低（已有 VPN 的话） |
| 方案 D：企微文件上传 | 把文档先上传到企微文件存储，再返回企微内部链接 | 高 |

**建议**：优先调研公司是否已有公网入口或 VPN 方案，再选择对应方案。
