# 全新服务器部署清单：Dify + RAGFlow + wecom-bot v3

## 一、前提

- 服务器已安装 Docker 和 Docker Compose
- Dify 和 RAGFlow 已通过各自的 docker-compose 部署运行
- 两者在同一台服务器上

---

## 二、Dify 侧需要改的

### 2.1 安装 wecom-bot 插件

从 Dify 市场安装，插件设置中配置三个必填项：

| 配置项 | 说明 |
|--------|------|
| Token | 企业微信回调 Token |
| Encoding AES Key | 企业微信回调加密 Key |
| App | 选择要绑定的 Dify 应用 |

### 2.2 部署 v3 插件代码

将 `wecom_v3.b64` 上传到服务器，部署到插件目录：

```bash
cd /opt/dify/docker/volumes/plugin_daemon/cwd/langgenius/wecom-bot-0.0.6@*/endpoints/
sudo base64 -d /tmp/wecom_v3.b64 > wecom_message.py
sudo docker restart dify-plugin_daemon-1
```

### 2.3 配置 API Key

在企业微信机器人对话框发送：
```
/fbkey app-xxxxxxxxxxxx
```

API Key 从 Dify Web → 对应 App → API 访问 → API 密钥 获取。

> Dify 插件设置页的 `RAGFlow 访问地址` 和 `Dify 服务 API 密钥` 可能因 schema 缓存不显示，不影响使用（代码已硬编码 + /fbkey 兜底）。

## 三、RAGFlow 侧需要改的

### 3.1 nginx 下载路由持久化

**原理**：RAGFlow 的 `entrypoint.sh` 每次启动从模板 `ragflow.conf.python` 执行 `cp -f` 覆盖 nginx 配置。把宿主机修改过的模板挂载进去，覆盖操作自然包含 doc-download。

**步骤**：

1. 导出容器内模板到宿主机：
```bash
sudo docker exec ragflow-ragflow-cpu-1 cat /etc/nginx/conf.d/ragflow.conf.python > /opt/ragflow/docker/nginx/ragflow.conf.python
```

2. 编辑 `/opt/ragflow/docker/nginx/ragflow.conf.python`，在 `location ~ ^/(v1|api) {` 前面插入：
```nginx
    location ~ ^/doc-download/([^/]+)$ {
        proxy_set_header Authorization "Bearer ragflow-你的API Token";
        rewrite ^/doc-download/(.*)$ /api/v1/documents/$1 break;
        proxy_pass http://localhost:9380;
        proxy_pass_header Content-Disposition;
        include proxy.conf;
    }
```

3. 编辑 `/opt/ragflow/docker/docker-compose.yml`，在 `ragflow-cpu` 服务的 `volumes` 下新增一行：
```yaml
      - ./nginx/ragflow.conf.python:/etc/nginx/conf.d/ragflow.conf.python
```

4. 重建容器：
```bash
cd /opt/ragflow/docker && sudo docker compose down && sudo docker compose up -d
```

5. 验证：
```bash
curl -sI "http://服务器IP:8088/doc-download/任意文档ID" | head -3
# 返回 HTTP/1.1 200
```

## 四、网络互通

```bash
# RAGFlow 加入 Dify 网络
docker network connect docker_default ragflow-ragflow-cpu-1

# Redis 网络别名
docker network disconnect docker_default docker-redis-1 2>/dev/null
docker network connect --alias redis docker_default docker-redis-1
```

## 五、推荐但不必须

### 5.1 Infinity 端口加固

编辑 `docker-compose-base.yml`：
```yaml
# 改前
      - ${INFINITY_CS_PORT:-5432}:5432
# 改后
      - 127.0.0.1:${INFINITY_CS_PORT:-5432}:5432
```

防止外部扫描导致 Infinity 假死。

### 5.2 Dify 并发调优

Dify 的 `api.env`：
```bash
SERVER_WORKER_AMOUNT=2          # 原 1
SERVER_WORKER_CONNECTIONS=50    # 原 10
```

## 七、部署前容易忽略的点

1. **v3 代码里的两个硬编码要先改**：生成 `wecom_v3.b64` 前，确认 `DIFY_API_BASE` 和 `RAGFLOW_BASE_URL` 跟新服务器一致
2. **插件目录哈希值每台机器不同**：`wecom-bot-0.0.6@xxx` 的 `@` 后面那串是安装时生成的，用 `find /opt -name "wecom_message.py" -path "*/wecom-bot*"` 找到实际路径
3. **RAGFlow API Token 每台不同**：nginx 里写的 `Bearer ragflow-xxx` 需从新服务器的 RAGFlow Web → API Token 页面获取
4. **docker-compose.yml 挂载加到 ragflow-cpu 服务下**：不要错加到 ragflow-gpu 或注释行
5. **网络互通的容器名可能不同**：用 `docker ps --format "{{.Names}}"` 确认实际容器名
6. **插件部署后要确认启动成功**：`docker logs dify-plugin_daemon-1 --tail 20 | grep "wecom-bot.*ready"`
